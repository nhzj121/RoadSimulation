## 1. 系统概述

### 1.1 功能定位

加工链系统是一个用于管理**跨 POI（Point of Interest）加工流程**的核心业务模块。它支持定义从原材料到成品的完整加工链条，管理加工订单的执行，并追踪每个工序的加工进度。

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| **加工链 (ProcessingChain)** | 定义完整的跨 POI 加工流程，包含多个有序工序 |
| **工序 (ProcessingStage)** | 加工链中的单个加工点，定义在特定 POI 进行的处理操作 |
| **加工订单 (ProcessingOrder)** | 一次完整的跨 POI 加工任务实例 |
| **加工任务 (ProcessingTask)** | 单个工序的执行实例，隶属于加工订单 |

### 1.3 业务流程

```
原材料 → [工序 1: POI_A] → [工序 2: POI_B] → [工序 3: POI_C] → 成品
         ↓                    ↓                    ↓
      加工任务 1            加工任务 2            加工任务 3
```

### 1.4 应用场景

- **钢铁→家具加工链**: 铁矿开采 → 钢铁冶炼 → 钢材加工 → 家具制造
- **农产品加工链**: 农作物种植 → 初加工 → 深加工 → 包装销售
- **化工加工链**: 原料处理 → 化学反应 → 产品精制 → 成品包装

## 4. 核心业务逻辑

### 4.1 加工链管理

#### 4.1.1 创建加工链

```java
@Transactional
public ProcessingChain createChain(ProcessingChain chain) {
    // 1. 检查编码唯一性
    if (processingChainRepository.existsByChainCode(chain.getChainCode())) {
        throw new IllegalArgumentException("加工链编码已存在");
    }
    
    // 2. 计算总加工时间
    int totalTime = chain.getStages().stream()
            .mapToInt(ProcessingStage::getProcessingTimeMinutes)
            .sum();
    chain.setTotalProcessingTimeMinutes(totalTime);
    
    // 3. 保存加工链
    return processingChainRepository.save(chain);
}
```

**业务规则**:
- 加工链编码必须唯一
- 自动计算总加工时间（各工序时间之和）
- 工序与加工链双向关联

#### 4.1.2 添加工序

```java
@Transactional
public ProcessingStage createStage(Long chainId, ProcessingStage stage) {
    ProcessingChain chain = processingChainRepository.findById(chainId)
            .orElseThrow(() -> new RuntimeException("加工链不存在"));
    
    stage.setProcessingChain(chain);
    ProcessingStage saved = processingStageRepository.save(stage);
    
    // 重新计算总时间
    int totalTime = chain.getStages().stream()
            .mapToInt(ProcessingStage::getProcessingTimeMinutes)
            .sum();
    chain.setTotalProcessingTimeMinutes(totalTime);
    processingChainRepository.save(chain);
    
    return saved;
}
```

### 4.2 加工订单管理

#### 4.2.1 创建加工订单

```java
@Transactional
public ProcessingOrder createProcessingOrder(Long chainId, Double inputWeight, String createdBy) {
    // 1. 验证加工链
    ProcessingChain chain = processingChainRepository.findById(chainId)
            .orElseThrow(() -> new RuntimeException("加工链不存在"));
    
    if (chain.getStatus() != ProcessingChain.ChainStatus.ACTIVE) {
        throw new IllegalStateException("加工链未激活");
    }
    
    // 2. 创建订单
    ProcessingOrder order = new ProcessingOrder();
    order.setOrderNo(generateOrderNo());
    order.setProcessingChain(chain);
    order.setStatus(ProcessingOrder.OrderStatus.PENDING);
    order.setInputWeight(inputWeight);
    order.setExpectedOutputWeight(inputWeight * chain.getYieldRate());
    
    // 3. 为每个工序创建加工任务
    for (ProcessingStage stage : chain.getStages()) {
        ProcessingTask task = new ProcessingTask();
        task.setProcessingOrder(order);
        task.setStage(stage);
        task.setStatus(ProcessingTask.TaskStatus.WAITING);
        task.setProcessedWeight(inputWeight);
        task.setProgressPercent(0);
        order.addTask(task);
    }
    
    // 4. 计算预期完成时间
    int totalMinutes = chain.getTotalProcessingTimeMinutes();
    order.setExpectedFinishTime(LocalDateTime.now().plusMinutes(totalMinutes));
    
    return processingOrderRepository.save(order);
}
```

**业务规则**:
- 只有激活状态的加工链可接收订单
- 订单号自动生成（含时间戳）
- 自动创建所有工序的加工任务
- 根据产出率计算预期输出重量

#### 4.2.2 开始加工

```java
@Transactional
public void startProcessing(Long orderId) {
    ProcessingOrder order = processingOrderRepository.findById(orderId)
            .orElseThrow(() -> new RuntimeException("加工订单不存在"));
    
    if (order.getStatus() != ProcessingOrder.OrderStatus.PENDING) {
        throw new IllegalStateException("订单状态不允许开始加工");
    }
    
    // 更新订单状态
    order.setStatus(ProcessingOrder.OrderStatus.IN_PROCESS);
    order.setStartTime(LocalDateTime.now());
    
    // 启动第一道工序
    ProcessingTask firstTask = order.getTasks().stream()
            .filter(t -> t.getStage().getStageOrder() == 1)
            .findFirst()
            .orElseThrow(() -> new RuntimeException("未找到第一道工序"));
    
    firstTask.setStatus(ProcessingTask.TaskStatus.PROCESSING);
    firstTask.setStartTime(LocalDateTime.now());
    
    processingOrderRepository.save(order);
}
```

### 4.3 加工进度追踪

#### 4.3.1 更新加工进度

```java
@Transactional
public void updateProcessingProgress(LocalDateTime simNow, int minutesPerLoop) {
    // 查询所有进行中的加工订单
    List<ProcessingOrder> orders = processingOrderRepository.findByStatus(
            ProcessingOrder.OrderStatus.IN_PROCESS);
    
    for (ProcessingOrder order : orders) {
        processOrder(order, simNow, minutesPerLoop);
    }
}

private void processOrder(ProcessingOrder order, LocalDateTime simNow, int minutesPerLoop) {
    for (ProcessingTask task : order.getTasks()) {
        if (task.getStatus() == ProcessingTask.TaskStatus.PROCESSING) {
            // 计算加工进度
            Duration elapsed = Duration.between(task.getStartTime(), simNow);
            int totalMinutes = task.getStage().getProcessingTimeMinutes();
            int progress = Math.min(100, (int)(elapsed.toMinutes() * 100 / totalMinutes));
            task.setProgressPercent(progress);
            
            // 工序完成处理
            if (progress >= 100) {
                completeTask(task, simNow);
                startNextTask(order, task, simNow);
            }
            break; // 只处理一个进行中的任务
        }
    }
    checkAndCompleteOrder(order, simNow);
}
```

**进度计算逻辑**:
```
进度百分比 = min(100, (当前时间 - 开始时间) / 工序总时间 × 100)
```

#### 4.3.2 工序流转

```java
private void completeTask(ProcessingTask task, LocalDateTime simNow) {
    task.setStatus(ProcessingTask.TaskStatus.COMPLETED);
    task.setEndTime(simNow);
    task.setProgressPercent(100);
    processingTaskRepository.save(task);
}

private void startNextTask(ProcessingOrder order, ProcessingTask currentTask, LocalDateTime simNow) {
    int currentOrder = currentTask.getStage().getStageOrder();
    
    // 查找下一道工序
    ProcessingTask nextTask = order.getTasks().stream()
            .filter(t -> t.getStage().getStageOrder() == currentOrder + 1)
            .findFirst()
            .orElse(null);
    
    if (nextTask != null) {
        nextTask.setStatus(ProcessingTask.TaskStatus.PROCESSING);
        nextTask.setStartTime(simNow);
        createTransportAssignment(currentTask, nextTask, simNow);
    }
}
```

**工序流转规则**:
1. 当前工序完成后，自动启动下一道工序
2. 工序间创建运输任务（Assignment）
3. 最后一道工序完成后，订单状态变为 COMPLETED

### 4.4 订单取消

```java
@Transactional
public void cancelProcessing(Long orderId) {
    ProcessingOrder order = processingOrderRepository.findById(orderId)
            .orElseThrow(() -> new RuntimeException("加工订单不存在"));
    
    if (order.getStatus() == OrderStatus.COMPLETED || 
        order.getStatus() == OrderStatus.CANCELLED) {
        throw new IllegalStateException("订单已完成或已取消");
    }
    
    order.setStatus(OrderStatus.CANCELLED);
    order.setActualFinishTime(LocalDateTime.now());
    
    // 取消所有未完成的任务
    for (ProcessingTask task : order.getTasks()) {
        if (task.getStatus() != TaskStatus.COMPLETED) {
            task.setStatus(TaskStatus.CANCELLED);
        }
    }
    
    processingOrderRepository.save(order);
}
```

## 6. 数据库设计

### 6.1 表结构

#### 6.1.1 processing_chain (加工链主表)

```sql
CREATE TABLE processing_chain (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    chain_code VARCHAR(50) NOT NULL UNIQUE,
    chain_name VARCHAR(100) NOT NULL,
    status VARCHAR(20) DEFAULT 'ACTIVE',
    description VARCHAR(500),
    total_processing_time_minutes INT,
    input_weight_per_cycle DECIMAL(10,2),
    output_weight_per_cycle DECIMAL(10,2),
    yield_rate DECIMAL(5,4) DEFAULT 0.95,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_chain_status (status),
    INDEX idx_chain_code (chain_code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 6.1.2 processing_stage (加工工序表)

```sql
CREATE TABLE processing_stage (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    chain_id BIGINT NOT NULL,
    stage_order INT NOT NULL,
    stage_name VARCHAR(100) NOT NULL,
    description VARCHAR(500),
    poi_id BIGINT NOT NULL,
    input_goods_id BIGINT,
    input_goods_sku VARCHAR(50),
    input_weight_ratio DECIMAL(10,4) DEFAULT 1.0,
    output_goods_id BIGINT,
    output_goods_sku VARCHAR(50),
    output_weight_ratio DECIMAL(10,4) DEFAULT 1.0,
    processing_time_minutes INT NOT NULL DEFAULT 60,
    max_capacity_per_cycle DECIMAL(10,2),
    min_batch_size DECIMAL(10,2),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (chain_id) REFERENCES processing_chain(id) ON DELETE CASCADE,
    FOREIGN KEY (poi_id) REFERENCES POI(id),
    FOREIGN KEY (input_goods_id) REFERENCES goods(id),
    FOREIGN KEY (output_goods_id) REFERENCES goods(id),
    INDEX idx_stage_chain (chain_id),
    INDEX idx_stage_poi (poi_id),
    INDEX idx_stage_order (chain_id, stage_order)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

```sql
    FOREIGN KEY (chain_id) REFERENCES processing_chain(id),
    FOREIGN KEY (input_goods_id) REFERENCES goods(id),
    FOREIGN KEY (output_goods_id) REFERENCES goods(id),
    INDEX idx_order_chain (chain_id),
    INDEX idx_order_status (status),
    INDEX idx_order_no (order_no)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 6.1.4 processing_task (加工任务表)

```sql
CREATE TABLE processing_task (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    order_id BIGINT NOT NULL,
    stage_id BIGINT NOT NULL,
    status VARCHAR(20) NOT NULL DEFAULT 'WAITING',
    progress_percent INT DEFAULT 0,
    processed_weight DECIMAL(10,2),
    start_time DATETIME,
    end_time DATETIME,
    inbound_assignment_id BIGINT,
    outbound_assignment_id BIGINT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (order_id) REFERENCES processing_order(id) ON DELETE CASCADE,
    FOREIGN KEY (stage_id) REFERENCES processing_stage(id),
    FOREIGN KEY (inbound_assignment_id) REFERENCES assignment(id),
    FOREIGN KEY (outbound_assignment_id) REFERENCES assignment(id),
    INDEX idx_task_order (order_id),
    INDEX idx_task_stage (stage_id),
    INDEX idx_task_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

```java
ProcessingChain chain = new ProcessingChain();

// 2. 准备 POI 和货物
```

### 8.4 前端调用示例 (TypeScript)

```typescript
// 创建加工订单
async createProcessingOrder(chainId: number, inputWeight: number): Promise<ProcessingOrderDTO> {
  const response = await fetch(`/api/processing-chain/${chainId}/order`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ inputWeight, createdBy: 'user_001' })
  });
  return response.json();
}

async startProcessing(orderId: number): Promise<void> {
  await fetch(`/api/processing-chain/order/${orderId}/start`, {
    method: 'POST'
  });
}

// 查询订单状态
async getOrderStatus(orderId: number): Promise<ProcessingOrderStatusDTO> {
  const response = await fetch(`/api/processing-chain/order/${orderId}/status`);
  return response.json();
}

// 轮询订单状态
async pollOrderStatus(orderId: number): Promise<void> {
  const interval = setInterval(async () => {
    const status = await getOrderStatus(orderId);
    console.log(`进度：${status.overallProgress}%`);
    
    if (status.overallProgress >= 100) {
      clearInterval(interval);
      console.log('加工完成!');
    }
  }, 5000); // 每 5 秒轮询一次
}
```

### C. 注意事项

1. **事务管理**: 所有写操作使用 `@Transactional` 保证数据一致性
2. **订单号唯一性**: 订单号包含纳秒级时间戳避免重复
3. **级联删除**: 删除加工链会自动删除关联工序（ON DELETE CASCADE）
4. **进度计算**: 基于仿真时间而非系统时间
5. **工序流转**: 需要实现运输任务 (Assignment) 的创建逻辑
